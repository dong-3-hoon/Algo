### problem a
    api를 받아와서 읽고 정보를 추출하는 방법을 배울 수 있었습니다. 처음에는 api url을 얻고 어떤 파라미터를 입력해야 알맞은 api를 불러오는지 몰랐고, 이후 json파일이 복잡하게 구성돼있어서 정보 한 개를 추출하는데 조금 어려움을 겪었습니다.
### problem b
    problem a 에서 조금만 응용하면 되는 문제였기 때문에 어려움은 없었습니다.
### problem c
    list안의 dict에서 values를 기준으로 정렬하는데 어려움을 겪었습니다. 전처럼 for문 2-3중으로 구성하는 방법도 있었지만 코드가 길어지기 때문에 아침에 배운 lambda방법을 활용해 정렬을 시도하는데 시간이 걸렸고, 해결할 수 있었습니다.
### problem d
    주어진 파라미터를 문자열에 삽입해 정보를 찾아오는데 처음에는 문자열을 나누고 파라미터를 문자열로 변경해 합쳤는데 비효율적이라서 fstring을 활용해서 문제를 해결했습니다.
### problem e
    problem d와 비슷한 유형의 문제여서 어려움이 덜했습니다. 이후 for문을 잘못 써 여러 값이 출력됐지만 실수를 깨닫고 문제를 해결했습니다.
### problem f
    두 api참조 사이트를 선택하고 박스 오피스 순위, 영화이름을 입력하면 정보를 띄우게 만들었다. 기존에는 4개의 api 제공 사이트를 활용하려 했으나 naver는 양식이 달랐고, kmdb는 api발급이 1`2일 걸리기 때문에 2개만 활용했다. 발급은 받지 못했지만 각 사이트에서 api를 활용한 정보를 불러올 수 있는 양식을 학습할 수 있었다.
### 총평
    api를 다뤄서 웹의 정보를 직접적으로 가져오는 것이 재밌었습니다.
